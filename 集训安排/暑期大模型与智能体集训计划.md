# 暑期大模型与智能体集训计划

> 适用对象：管理科学与工程学科低年级研究生及准研究生
> 集训时长：6 周（另含第 0 周预备期）
> 组织方式：个人自学 + 每周二、五晚 19:30 腾讯会议线上检查
> 配套文档：《知识点学习大纲.md》（周二选讲的统一参照）

---

## 一、集训概述

### 1.1 集训目标

本次集训结束后，每位学生应达到：

1. **懂原理**：理解大模型的工作原理、能力边界与最新技术动态（推理模型、上下文工程、Harness Engineering、MCP 等），能用准确的术语讨论问题；
2. **会用工具**：熟练使用 Claude Code 等 AI 编程工具辅助科研与开发，养成"AI 协作"的工作习惯；
3. **能做开发**：能独立调用大模型 API，搭建包含工具调用、RAG 或多智能体的应用原型；
4. **衔接科研**：完成一个面向管理科学与工程研究场景的个人项目，为后续将大模型/智能体方法引入自己的研究方向打下基础。

### 1.2 组织方式

- **划定范围、个人自学**：知识范围由本计划和《知识点学习大纲》划定，学生**个人单独**完成每周的学习与实践任务，不分组。
- **双周检查**：每周二、五晚 19:30 腾讯会议，全员参加，共 12 次汇报：
  - **知识点选讲（每次会议都有）**：《知识点学习大纲》的 31 个条目按排期分配到 12 次会议（周二每晚 3-4 条、周五每晚 2 条）。**不设提前认领，每个学生都必须为全部条目准备选讲 PPT**，汇报人当晚现场抽签产生（不放回抽签，保证人人轮到），讲解不设时间限制、以讲透为准，导师与同学提问；
  - **周五预留实践展示**：选讲之后，演示本周动手任务/项目的运行效果，走读关键代码，汇报问题与进度。
- **统一仓库**：建立一个集训 GitHub（或 Gitee）组织仓库，每人一个目录，提交学习笔记与代码，所有产出公开互见。

### 1.3 算力与模型资源约定

本次集训**不涉及模型预训练与微调**（仅在第 1 周作为知识点了解原理），全部实践基于现成大模型产品与 API 完成：

| 用途 | 推荐选择 | 说明 |
|---|---|---|
| 日常对话/学习答疑 | 豆包、DeepSeek 网页版、Kimi | 免费、国内直连 |
| 编程实践主力 API | **DeepSeek API**、智谱 GLM API、通义千问 API | 国内直连、价格低（DeepSeek 每百万 token 几元钱，充 50 元足够全程实验） |
| AI 编程工具 | Claude Code（首选）、Cursor、Trae | Claude Code 需要 Anthropic 账号或兼容 API 中转；条件不具备的学生可用 Trae（国内直连）替代，但需对照学习 Claude Code 的概念 |
| 高质量对照实验 | Claude、ChatGPT | 视个人条件使用，用于体会模型间差异 |

> **费用提示**：建议每人准备 50-100 元 API 预算（以 DeepSeek 为主力可大幅压低成本）。注意为 API Key 设置用量上限，避免循环调用失控烧钱。

## 二、第 0 周：预备（集训开始前完成）

**目标：环境就绪、基础拉齐。**

**开营宣讲（安排在第 0 周内，时间另行通知）**：腾讯会议全员参加，导师宣讲集训计划、选讲与抽签规则、考核说明，并抽查下述环境准备清单的完成情况。

### 2.1 环境准备清单（人人必做）

- [ ] 安装 Python 3.11+（建议用 `uv` 或 Anaconda 管理环境）
- [ ] 安装 Git，注册 GitHub 账号，完成一次 clone → 修改 → commit → push 全流程
- [ ] 安装 VS Code
- [ ] 安装 Claude Code（条件不具备者安装 Trae 作为替代，但仍需阅读 Claude Code 文档）
- [ ] 注册并充值 DeepSeek API（或智谱 GLM），跑通官方文档里的第一个 API 调用示例
- [ ] 注册豆包/Kimi 等至少一个对话产品，日常学习中养成"先问 AI"的习惯
- [ ] 加入集训统一代码仓库，在自己名下目录提交一个 `hello.md` 确认权限

### 2.2 基础拉齐（按需自学，准研究生重点）

- **Python 速成**：B 站搜索"Python 速成"任选一套 4-8 小时课程；或直接让大模型当家教，边问边练。重点：函数、字典/列表、虚拟环境、pip 安装包、读写文件、调用 HTTP API（requests）。
- **命令行基础**：cd / ls(dir) / 运行脚本 / 环境变量，Windows 用户了解 PowerShell 基本操作。
- **Git 基础**：clone、add、commit、push、分支的概念即可。
- **检验标准**：能在自己电脑上新建虚拟环境、安装 openai 库、用 DeepSeek API 跑通一个对话脚本——做不到的，第 1 周周五前必须补齐。

## 三、六周学习安排

> 每周自学当周排期的知识点条目（第 1-5 周每周 5 条、第 6 周 6 条，编号对应《知识点学习大纲.md》），与第四节的选讲排期完全一致：**周二会议前学完并备好当晚条目的选讲 PPT，周五会议前学完当周全部条目**。每周另有**选学**内容（学有余力者）、**动手任务**（周五展示）与**过关标准**（导师检查依据）。

### 第 1 周：大模型如何工作（条目 1.1-1.5）

**必学**
- 1.1 Transformer 架构直觉：注意力机制在做什么（不要求数学推导）
- 1.2 Token 化与上下文窗口：模型"看到"的世界是什么样的
- 1.3 预训练：下一词预测、训练数据与 Scaling Laws
- 1.4 后训练：SFT / RLHF / RLVR，模型的"性格"从哪来
- 1.5 推理模型与测试时计算：OpenAI o 系列、DeepSeek-R1 为什么是范式变化

**选学**
- 《Attention Is All You Need》原文；MoE 架构直觉

**动手任务（周五展示）**
- 选定一个与自己研究方向相关的问题（如"平台定价策略综述"），用 2-3 个不同模型（如 DeepSeek、豆包、Claude/ChatGPT）并对照"普通模式 vs 深度思考模式"做实验，整理成对比笔记并展示：模型间差异在哪、推理模式带来多大提升、发现了哪些可疑的幻觉。

**过关标准**
- 能向非专业同学讲清楚"大模型是怎么训练出来的"全流程；能解释为什么推理模型在数学/代码任务上更强；对照实验有方法、有记录、有结论。

### 第 2 周：模型生态、提示工程与 API 入门（条目 1.6-1.8、2.1-2.2）

**必学**
- 1.6 国内外大模型生态图谱：各家定位、开源与闭源格局、科研选型方法
- 1.7 提示工程系统方法：角色设定、少样本示例、思维链、输出格式约束
- 1.8 大模型的局限：幻觉、知识截止、谄媚性，以及科研使用红线
- 2.1 API 基础：messages 格式（system/user/assistant）、温度等参数、流式输出
- 2.2 结构化输出（JSON mode）与 Function Calling / Tool Use：让模型"动手"的机制

**选学**
- DeepSeek-V3 / R1 技术报告细读

**动手任务（周五展示）**
- 用 DeepSeek（或智谱）API 写一个**带工具调用的命令行问答程序**：至少实现 2 个工具（如"查询本地 CSV 数据"和"做数学计算"），模型自主决定何时调用工具，系统提示用本周所学提示工程方法打磨。展示运行效果并走读"模型返回工具调用 → 程序执行 → 结果回传"的代码循环。

**过关标准**
- 能徒手画出 function calling 的完整消息流转图；工具调用循环是自己写的（不是框架黑盒），能回答导师对任意一行代码的提问；能演示一个自己抓到的幻觉案例及核验方法。

### 第 3 周：上下文工程、RAG 与评测（条目 2.3-2.7）

**必学**
- 2.3 上下文工程：为什么业界已从"提示工程"升级到"上下文工程"——把正确的信息以正确的形式在正确的时机放进上下文
- 2.4 Embedding 与向量检索的原理
- 2.5 RAG（检索增强生成）：分块、检索、重排、生成的完整链路，以及它适合/不适合解决什么问题
- 2.6 评测基础：怎么科学地判断大模型输出的好坏，LLM-as-Judge 的用法与陷阱
- 2.7 Token 经济学：API 计费方式、prompt caching、控制成本的工程手段

**选学**
- 长上下文 vs RAG 的取舍之争；rerank 模型；Anthropic《Effective context engineering for AI agents》原文精读

**动手任务（周五展示）**
- 搭一个**最小 RAG 问答 demo**：导入 3-5 篇自己方向的文献，实现"分块 → 向量化 → 检索 → 带引用回答"的完整链路，并用 5-10 个自拟问题做一轮小评测（人工判分或 LLM-as-Judge）。展示问答效果，并分析至少一个检索出错的案例。

**过关标准**
- 能说清 RAG 每一环节可能出错的地方并结合自己的 demo 举例；能解释自己的小评测怎么设计、结论是否可信；能估算 demo 一次问答的 token 成本。

### 第 4 周：AI 编程工具与 Claude Code（条目 3.1-3.5）

**必学**
- 3.1 AI 编程范式演进：代码补全 → 对话式 → 智能体编程（Agentic Coding），各阶段的本质区别
- 3.2 Claude Code 核心工作流：安装配置、提需求的正确方式、CLAUDE.md（项目记忆）的写法
- 3.3 Claude Code 进阶（上）：Plan Mode（先规划后执行）、Slash Commands、Skills（技能包）
- 3.4 Claude Code 进阶（下）：Hooks（钩子）、Subagents（子智能体）、MCP 服务接入
- 3.5 同类工具的格局与取舍：Cursor、GitHub Copilot、国内的 Trae / 通义灵码

**选学**
- Anthropic《Claude Code: Best practices for agentic coding》；Claude Agent SDK 文档（为第 5-6 周铺垫）

**动手任务（周五展示）**
- 用 Claude Code（或替代工具）**从零完成一个与研究相关的小工具**并提交 GitHub 仓库，例如：问卷数据清洗与统计脚本、文献 BibTeX 整理器、实验数据可视化面板。要求：全程记录你与 AI 的协作过程（哪些是 AI 写的、你审查改正了什么、CLAUDE.md 怎么写的），展示时讲"协作复盘"而不只是讲功能。

**过关标准**
- 工具能现场运行；能说出 Plan Mode / CLAUDE.md / Subagent 分别解决什么问题；复盘中能举出至少一个"AI 写错了、我是怎么发现并纠正的"实例。

### 第 5 周：智能体与 Harness Engineering + 项目开题（条目 3.6-3.7、4.1-4.3）★ 技术制高点（上）

**必学**
- 3.6 "Vibe Coding"的能力与边界：什么任务可以放手让 AI 写，什么任务必须人盯着
- 3.7 AI 辅助科研编程的规范：可复现性、代码审查习惯、学术诚信边界
- 4.1 智能体的本质与 Agent 循环（感知 → 思考 → 工具调用 → 观察 → 迭代）、ReAct 模式
- 4.2 Harness Engineering（上）：**Agent = Model + Harness（模型 + 线束）**——模型负责推理、线束负责行动，harness 的组件构成，为什么业界共识是"产品护城河在线束层而不在模型层"
- 4.3 Harness Engineering（下）：上下文管理（压缩/记忆/子智能体隔离）、结果验证与长时运行智能体设计（含 Anthropic 2026 年 6 月发布的动态工作流"A harness for every task"——让智能体为每个任务现场构建专属线束）

**选学**
- Anthropic《Building Effective Agents》《Effective harnesses for long-running agents》原文精读

**项目任务（本周起学习与项目并行，周五展示）**
- 周二会议：**人人开题**（5 分钟/人）：选题、研究/应用场景、技术方案、两周里程碑
- 周五会议：原型进度展示，导师逐人把关技术路线——原型中须体现所学（至少包含一个自己写的 agent 循环或工具调用环节）

**过关标准**
- 能不看资料讲清"harness 包含哪些组件、分别解决什么问题"；开题的技术方案能说清每一步用什么机制实现；周五有可运行的原型雏形。

### 第 6 周：MCP、智能体框架与多智能体 + 结题（条目 4.4-4.9）★ 技术制高点（下）

**必学**
- 4.4 **MCP（Model Context Protocol）**：智能体连接工具与数据的行业标准协议，最简 MCP server 的写法与接入
- 4.5 Claude Agent SDK：把 Claude Code 同款 harness 拿来构建自己的智能体
- 4.6 智能体框架概览与选型：Claude Agent SDK、LangGraph、CrewAI 的设计哲学差异
- 4.7 多智能体系统：编排模式（编排者-工作者、辩论、流水线）、A2A 协议、典型失效模式（错误级联、上下文撕裂）
- 4.8 智能体方法在管理科学研究中的应用前沿：LLM 多智能体仿真做社会/管理实验（Generative Agents 一脉的研究）
- 4.9 个人 AI 智能体现象：**OpenClaw** 与 **Hermes Agent**——2026 年爆发的开源自托管个人智能体（接入消息平台、持久记忆、自动技能创建），用 harness 视角与 Claude Code 对比（**了解级，不要求部署**）

**选学**
- A2A 协议规范；与自己项目相关的近两年文献 2-3 篇

**项目任务**
- 周二会议：项目中期演示 + 共性疑难会诊；鼓励把本周所学反哺项目（如为项目接入自写 MCP server、用框架重构编排逻辑）
- 周五会议：**结题答辩**（每人 10 分钟展示 + 5 分钟问答）

**过关标准**
- 能说出至少两种多智能体失效模式及对应的工程对策；结题项目达到下方"项目要求"；答辩中能回答"为什么这样设计 harness/流程"的追问。

### 个人项目实战（第 5-6 周，与学习并行）

**目标：每人独立完成一个面向管理科学与工程研究场景的项目，规模控制在 2 周内可完成。**

**项目菜单**（每人选一，也可自拟报导师批准；★=入门档，★★=进阶档）

| # | 项目 | 难度 | 说明 |
|---|---|---|---|
| 1 | LLM 多智能体管理实验仿真 | ★★ | 用多个 LLM agent 模拟拍卖、谈判或供应链博弈场景，设计实验、记录 agent 行为、分析结果。可直接衔接行为运营/实验经济学方向的科研 |
| 2 | 智能文献综述助手 | ★ | RAG + 智能体：导入本方向 30-100 篇文献，实现"按主题问答、自动归纳研究脉络、生成带引用的综述初稿" |
| 3 | 企业数据分析智能体 | ★ | 自然语言 → 自动生成查询/分析代码 → 图表 → 管理建议，基于一份公开企业数据集（如电商订单、生产数据） |
| 4 | AI 辅助运筹优化智能体 | ★★ | LLM 理解自然语言描述的排产/选址/路径问题 → 自动建模并调用求解器（OR-Tools/Gurobi）→ 解释求解结果 |
| 5 | 学术写作与评审智能体工作流 | ★ | 多步智能体流水线：初稿分析 → 按期刊规范逐项审查 → 生成修改建议，体现 harness 设计思想 |

**节奏安排**：开题（第 5 周周二）→ 原型展示（第 5 周周五）→ 中期检查（第 6 周周二）→ 结题答辩（第 6 周周五，人数多可分两场或延长当晚时间），已纳入上方第 5、6 周的"项目任务"。

**项目要求**
- GitHub 仓库：README（含运行说明）、可运行代码、必要的实验记录
- 结题报告（3-5 页，按学术规范）：问题定义、方法与系统设计（须包含一张系统/harness 架构图）、结果与演示、局限与后续研究设想
- 鼓励但不要求：把项目发展为后续课题或论文的预研

## 五、考核与产出要求

**每人须完成：**

1. **学习笔记**：每周一篇 Markdown 笔记提交到统一仓库（截止周日 24:00），记录本周必学内容的理解、动手任务过程与踩坑；
2. **知识点选讲**：为《知识点学习大纲》全部 31 个条目准备选讲 PPT，并在每次会议当天 18:00 前提交当晚排期条目的 PPT（无论是否被抽中，提交即准备凭证）；被抽中时上台讲解；
3. **每周动手任务**：第 1-4 周各一项，周五展示或提交；
4. **个人项目**：第 5-6 周完成，含代码仓库 + 结题报告 + 答辩。

**评价构成（建议）：**

| 维度 | 权重 | 依据 |
|---|---|---|
| 知识掌握 | 30% | 抽中时的选讲质量 + 选讲 PPT 提交完成度 + 答问表现 + 学习笔记 |
| 动手任务 | 30% | 第 1-4 周四次任务的完成度与代码质量 |
| 项目成果 | 40% | 结题答辩 + 仓库与报告质量 |

**红线**：学习笔记与代码不允许整段粘贴 AI 输出冒充自己的理解——可以用 AI 辅助，但必须能答上导师对内容的任意追问；选讲 PPT 必须包含自己跑通的示例。

## 七、给学生的三条学习建议

1. **把大模型当成全天候导师**：本次集训的最佳学习方法本身就是用大模型学大模型——概念不懂先问 AI、代码报错先贴给 AI，把"AI 协作"练成肌肉记忆，这正是集训目标之一。
2. **一切以跑通为准**：看十篇文章不如跑通一个 demo。每个知识点都问自己"我能演示它吗"，周二选讲必须带自己跑通的例子，这是硬性规定。
3. **始终连接研究方向**：每周问自己一次"这个技术能为我的研究做什么"——把想法记进周记，第 5 周开题时它们就是现成的选题来源。
