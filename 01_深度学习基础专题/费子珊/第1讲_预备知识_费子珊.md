## 2.1 数据操作

### 入门

深度学习里，所有数据都以"张量（tensor）"表示。张量就是一个 n 维数组，是深度学习中的"通用货币"——模型的输入、输出、参数，全都以张量的形式存在。

### 不同维度的张量

| 维度 | 数学名称 | 示例 | 代码 |
|------|---------|------|------|
| 0 | 标量（scalar） | 一个数，如 `3.14` | `torch.tensor(3.14)` |
| 1 | 向量（vector） | 一组数，如 `[1.0, 2.0, 3.0]` | `torch.arange(4)` |
| 2 | 矩阵（matrix） | 二维表格，如一张 28×28 灰度图 | `torch.randn(28, 28)` |
| 3 | 三维张量 | 一张彩色图片 (通道, 高, 宽) | `torch.randn(3, 224, 224)` |
| 4 | 四维张量 | 一批图片 (批量, 通道, 高, 宽) | `torch.randn(32, 3, 224, 224)` |

张量有两个重要的属性：
- **形状（shape）**：每个维度有多大
- **数据类型（dtype）**：每个元素是什么类型（如 `torch.float32`）

```python
import torch

x = torch.arange(12)     # 创建一个包含 0~11 的行向量
x.shape                  # torch.Size([12])
x.numel()                # 12（元素总数）
X = x.reshape(3, 4)      # 改成 3 行 4 列
X = x.reshape(-1, 4)     # -1 自动推导行数
X = x.reshape(3, -1)     # -1 自动推导列数
```

常用的张量创建函数：

```python
torch.zeros((2, 3, 4))                            # 全 0
torch.ones((2, 3, 4))                             # 全 1
torch.randn(3, 4)                                  # 标准正态分布随机
torch.tensor([[2, 1, 4, 3], [1, 2, 3, 4], [4, 3, 2, 1]])  # 直接给值
```

### 运算符

```python
x = torch.tensor([1.0, 2, 4, 8])
y = torch.tensor([2, 2, 2, 2])
x + y, x - y, x * y, x / y, x ** y      # 逐元素运算
torch.exp(x)                              # 逐元素指数

# 连结（concatenation）
X = torch.arange(12, dtype=torch.float32).reshape((3, 4))
Y = torch.tensor([[2.0, 1, 4, 3], [1, 2, 3, 4], [4, 3, 2, 1]])
torch.cat((X, Y), dim=0)   # 沿行 (6, 4)
torch.cat((X, Y), dim=1)   # 沿列 (3, 8)

# 比较
X == Y                     # 逐元素判断是否相等

# 求和
X.sum()                    # 所有元素求和
```

### 广播机制

形状不同的张量在做逐元素运算时，会通过"广播（broadcasting）"自动扩展：

1. 从最后一维向前比对，某维大小为 1 则复制到与对方一致
2. 某维在对方中不存在，则插入大小为 1 的维度再复制

![alt text](a14b5573ebfcef76d51f0e176c9c783.jpg)

```python
a = torch.arange(3).reshape((3, 1))
b = torch.arange(2).reshape((1, 2))
a + b    # shape: (3, 2)
```
![alt text](image-2.png)

### 索引和切片

```python
X[-1]        # 最后一行
X[1:3]       # 第 2、3 行
X[1, 2] = 9  # 修改单个元素
X[0:2, :] = 12  # 修改前两行的所有列
```

### 节省内存

运算默认分配新内存，这在大批量训练时可能浪费显存：

```python
before = id(Y)
Y = Y + X
id(Y) == before     # False —— 分配了新内存

# 原地操作
Z = torch.zeros_like(Y)
Z[:] = X + Y        # 写到已有 Z 里，没有新内存分配

before = id(X)
X += Y              # 真正的原地操作
id(X) == before     # True
```

> 🔗 在训练循环中反复创建新张量会消耗大量内存，所以后面的优化器会用原地更新：`param -= lr * param.grad`。

### 转换为其他 Python 对象

```python
A = X.numpy()               # tensor → numpy
B = torch.tensor(A)         # numpy → tensor
a = torch.tensor([3.5])
a.item()                    # 3.5 —— 将单元素张量转为 Python 标量
```

### 降维

求和操作默认对所有元素求和，也可以沿指定轴降维：

```python

A.sum(axis=1)        # 沿第 1 轴（列方向），保留行
A.mean()             # 均值
A.sum() / A.numel()  # 等价写法
```

用 `keepdims=True` 保持维度数不变，便于后续广播：

```python
sum_A = A.sum(axis=1, keepdims=True)    # shape: (5, 1) 而不是 (5,)
A / sum_A                                # 每行除以其总和（归一化）
```

沿某轴的**累积和**：

```python
A.cumsum(axis=0)     # 每行是前面所有行的累加
```

| 运算 | 函数 | 说明 |
|------|------|------|
| 向量点积 | `torch.dot(x, y)` | $\sum x_i y_i$，返回标量 |

## 2.4 微积分

### 导数和微分

导数 $f'(x)$ 衡量函数 $f(x)$ 在 $x$ 处的**瞬时变化率**。数值上可以用中心差分近似：

$$f'(x) \approx \frac{f(x+h) - f(x)}{h}$$

```python
def f(x):
    return 3 * x ** 2 - 4 * x

def numerical_lim(f, x, h):
    return (f(x + h) - f(x)) / h
```
![alt text](image-10.png)

h 越小，数值近似越接近解析导数。

![alt text](image-9.png)


### 偏导数

多元函数 $f(x_1, x_2, ..., x_n)$ 对某个变量的偏导数，就是把其他变量当作常数再求导，记为 $\frac{\partial f}{\partial x_i}$。

![alt text](image-11.png)

### 梯度

**梯度**是函数所有偏导数组成的向量：

$$\nabla f(\mathbf{x}) = \left[ \frac{\partial f}{\partial x_1}, \frac{\partial f}{\partial x_2}, ..., \frac{\partial f}{\partial x_n} \right]$$

梯度最重要的性质：**指向函数值增长最快的方向。**

```python
x = torch.tensor([3.0, 4.0], requires_grad=True)
y = (x ** 2).sum()        # f(x1,x2) = x1² + x2²，在(3,4)处值为25
y.backward()
x.grad                     # [6.0, 8.0]，指向(3,4)处的增长最快方向
```

> 🔗 深度学习的核心就是**梯度下降**：沿负梯度方向走一小步，使损失函数值减小。参数更新：$\theta \leftarrow \theta - \eta \cdot \nabla_\theta L$。

<!-- TODO: 插入梯度指向增长最快方向的示意图 -->

### 链式法则

复合函数 $y = f(g(x))$ 的导数：

$$\frac{dy}{dx} = \frac{dy}{dg} \cdot \frac{dg}{dx}$$

例如 $y = (x^2+1)^3$，拆成 $u = g(x) = x^2+1$，$y = f(u) = u^3$：

$$\frac{dy}{dx} = 3u^2 \cdot 2x = 6x(x^2+1)^2$$

```python
x = torch.tensor(2.0, requires_grad=True)
u = x ** 2 + 1
y = u ** 3
y.backward()
x.grad    # 300 = 6×2×(4+1)²
```

> 🔗 **这就是反向传播的地基。** 神经网络是一个巨大的复合函数，反向传播从输出层出发，逐层用链式法则把梯度传回每个参数。

## 2.5 自动微分

### 为什么需要自动微分？

| 方式 | 做法 | 问题 |
|------|------|------|
| 手动求导 | 手算公式写成代码 | 网络一改就得重算 |
| 数值微分 | $\frac{f(x+h)-f(x-h)}{2h}$ | 精度误差，太慢 |
| 符号微分 | 推导解析式再代入 | 表达式指数膨胀 |
| **自动微分** | 计算图 + 链式法则 | ✅ 精确，高效 |

### 一个简单的例子

```python
import torch

x = torch.arange(4.0, requires_grad=True)
y = 2 * torch.dot(x, x)     # y = 2(x0²+x1²+x2²+x3²) = 2×14 = 28
y.backward()
x.grad                       # [0., 4., 8., 12.]
x.grad == 4 * x              # True：∂y/∂xi = 4xi
```

`requires_grad=True` 开启梯度追踪，`backward()` 自动计算梯度并存入 `.grad`。

### 梯度累积（重要！）

PyTorch 默认**累积**梯度，每次 backward 后**必须手动清零**，否则梯度会叠加：

```python
x.grad.zero_()          # 清零
y = x.sum()             # y = x0+x1+x2+x3
y.backward()
x.grad                   # [1., 1., 1., 1.]
```

### 非标量变量的反向传播

`backward()` 只能对标量调用。如果 y 是向量，需要先转为标量：

```python
x.grad.zero_()
y = x * x               # y 是向量
y.sum().backward()      # ✅ 先求和再 backward
x.grad                   # [0., 2., 4., 6.] = 2*x
```
![alt text](image-12.png)

### 分离计算（detach）

`detach()` 把某个张量从计算图中"切"出来，其后的运算不再传播梯度：

```python
x.grad.zero_()
y = x * x
u = y.detach()           # u 被视为常量
z = u * x                # 对 autograd 来说 z = 常量 × x
z.sum().backward()
x.grad == u              # True（梯度 = u，而不是 3x²）
```

### Python 控制流也能求梯度

自动微分的强大之处：它能处理任意 Python 控制流（条件、循环）：

```python
def f(a):
    b = a * 2
    while b.norm() < 1000:
        b = b * 2
    if b.sum() > 0:
        c = b
    else:
        c = 100 * b
    return c

a = torch.randn(size=(), requires_grad=True)
d = f(a)
d.backward()
a.grad == d / a          # True
```

## 2.6 概率

### 基本概率论

概率论的核心思想：**用频率估计概率。** 公平骰子每个面概率为 1/6。

```python
from torch.distributions import multinomial

fair_probs = torch.ones([6]) / 6

# 掷 1 次
multinomial.Multinomial(1, fair_probs).sample()

# 掷 10 次
multinomial.Multinomial(10, fair_probs).sample()

# 掷 1000 次，用频率估计概率
counts = multinomial.Multinomial(1000, fair_probs).sample()
counts / 1000    # ≈ [0.167, 0.167, 0.167, 0.167, 0.167, 0.167]
```

**大数定律**：试验次数越多，频率越接近真实概率。

通过 500 组 × 每组 10 次实验，可以画出概率收敛曲线：

```python
counts = multinomial.Multinomial(10, fair_probs).sample((500,))
cum_counts = counts.cumsum(dim=0)
estimates = cum_counts / cum_counts.sum(dim=1, keepdims=True)
# 每条曲线逐渐收敛到 1/6 ≈ 0.167
```

### 联合概率与条件概率

- **联合概率** $P(A, B)$：A 和 B 同时发生
- **条件概率** $P(A \mid B) = P(A, B) / P(B)$
- **贝叶斯定理**：$P(A \mid B) = \frac{P(B \mid A) \cdot P(A)}{P(B)}$
- **独立性**：若 A、B 独立，则 $P(A, B) = P(A) \cdot P(B)$

### 期望与方差

- **期望** $\mathbb{E}[X]$：随机变量的加权平均
- **方差** $\text{Var}(X) = \mathbb{E}[(X - \mathbb{E}[X])^2]$：偏离期望的程度

```python
samples = torch.randn(10000)
samples.mean()    # ≈ 0（标准正态分布）
samples.var()     # ≈ 1
```

> 🔗 **与后续连接**：
> - 假设回归误差服从高斯分布 → 最大似然估计 → **MSE 损失**
> - 假设分类标签服从多项分布 → 最大似然估计 → **交叉熵损失**
