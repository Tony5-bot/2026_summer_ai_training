# 第二讲  线性神经网络

深度学习训练的目标是：持续迭代调整模型可学习参数，最小化损失函数，使模型在训练数据、未知测试数据上都尽可能输出贴合真实标签的预测，同时兼顾泛化、稳定等约束。

本节先从最简单的一类预测问题开始。

监督学习中的预测任务大致可分为两类：

- **回归问题**：预测一个连续数值，例如房价、温度、销量；
- **分类问题**：预测样本属于哪个类别，例如垃圾邮件或正常邮件、猫或狗。

无论是回归还是分类，核心都一样：

> 给定输入数据，模型需要学习一条从输入到输出的规则。

复杂的神经网络可以学习很复杂的规则，但在学习复杂模型之前，先看最简单最容易理解的规则：**把输入特征按不同重要程度加权后相加。**



### ==怎么进行模型拟合？==

已经有一些常用的拓扑结构（神经网络），运算逻辑，也就是提前设计好的数据处理流水线，用于处理不同复杂程度下的数据关系。

回归的各种标准工具中最简单的是线性回归。

假设我们要预测房价。每套房子有面积、房龄、距地铁距离等特征。直觉上：

- 面积越大，房价通常越高；
- 房龄越大，房价可能越低；
- 距地铁越远，房价可能越低。

最简单的做法是：为每个特征分配一个权重，再把所有特征的贡献加起来。

这就是**线性回归**。

线性回归是回归问题中最基础、最经典的模型，也是理解线性神经网络的起点。

线性回归假设：在其他因素不变时，每个特征对预测结果的影响可以用一个固定权重表示；最终预测值是各特征贡献的加权和，再加上一个整体偏移量。

需要注意：现实数据通常包含噪声、遗漏变量和随机因素，因此线性关系一般不是完全精确的。若进一步希望进行概率建模或统计推断，常会额外假设误差服从正态分布；但仅仅使用线性回归进行预测，并不要求噪声必须正态分布。

1.单样本展开形式

\(\hat{y} = w_1x_1+\dots +w_dx_d + b \tag{3.1.2}\)

2.单样本向量点积紧凑形式

\(\hat{y} = \boldsymbol w^\top \boldsymbol x + b \tag{3.1.3}\)

3.全批量数据集矩阵形式（n 个样本整体预测）

\(\hat{\bold{y}} =  \boldsymbol X \boldsymbol w + b\)

符号：

d：单个样本的特征维度数

\(\boldsymbol x\in\mathbb R^d\)：单个样本特征列向量

\(\boldsymbol w\in\mathbb R^d\)：权重列向量

b：标量偏置项

\(\boldsymbol X\in\mathbb R^{n\times d}\)：整体数据集矩阵，n 行对应 n 条样本，d 列对应 d 个特征

\(\hat{\boldsymbol y}\in\mathbb R^n\)：全部样本的预测结果向量



每一个输入特征 \(x_i\)，严格对应专属权重 \(w_i\)，一对一相乘后累加；名义上权重层在中间是隐藏层，但其实由于不带非线性激活，所以无法学习非线性规律，即使是多层线性网络，也等价于单层。

### ==线性神经网络是什么？==

线性回归可以看作最简单的神经网络：

- 输入层接收特征；
- 一个线性层计算“加权和加偏置”；
- 输出层直接给出预测结果。

<img src="C:\Users\s1339\AppData\Roaming\Typora\typora-user-images\image-20260708155133783.png" alt="image-20260708155133783" style="zoom: 80%;" />

它没有隐藏层，也没有非线性激活函数。

因此，线性回归只能学习线性关系。例如，它可以学习“面积每增加一定单位，房价平均增加多少”，但难以直接学习更复杂的规律，如“面积只有超过某个阈值后才明显影响房价”。

即使把多个纯线性层叠加起来，只要中间没有加入非线性激活函数，整个网络仍然等价于一个单层线性模型。

真正让深度神经网络具备复杂表达能力的，是后续要学习的非线性激活函数。

复杂神经网络不是凭空出现的，它是在“加权求和”这个最简单预测规则上，逐步加入多层结构、非线性激活和更复杂参数化后发展出来的。

### ==如何度量拟合的效果？==

模型已经能够根据输入给出预测值，但还需要判断：预测结果与真实标签之间到底差多少。
因此，我们定义损失函数（loss function），用来量化单个样本的预测值与真实值之间的差异。
对于回归任务，通常希望损失满足：
• 损失是非负的；
• 预测越准确，损失越小；
• 对平方误差这类损失而言，预测值与真实值完全相同时，损失为 0。
需要注意：训练后最小损失不一定能达到 0，因为真实数据可能包含噪声，或者线性模型本身无法完全表达真实规律。



线性回归中最常用的损失函数是**平方误差损失**。

1. 单个样本平方误差损失（带 1/2 系数）

\(l^{(i)}(\mathbf{w},b)=\frac12\left(\hat y^{(i)}-y^{(i)}\right)^2\)

2. 整个训练集平均损失（整体经验损失）

\(L(\mathbf{w},b)=\frac1n\sum_{i=1}^n l^{(i)}(\mathbf{w},b)=\frac1n\sum_{i=1}^n \frac12\left(\mathbf{w}^\top \mathbf{x}^{(i)}+b-y^{(i)}\right)^2\)      ===》等价于$\frac{1}{2}MSE$均方误差

3. 参数最优求解（最小化损失） 训练线性回归，就是寻找一组权重和偏置，使平均损失尽可能小：

\(\mathbf{w}^*,b^*=\mathop{\arg\min}_{\mathbf{w},b} L(\mathbf{w},b)\)

4. 线性回归闭式解析解

\(\mathbf w^* = \big(\mathbf X^\top \mathbf X\big)^{-1}\mathbf X^\top \mathbf y \tag{3.18}\)

### ==**在求解的时候，损失函数没有解析解怎么办？**==

首先优化问题，都要求导（有些优化问题不可导就用次梯度近似梯度之类的方法），导数可以给出优化的方向。

- 当问题有解析解的时候：

求导 → 令梯度\(\nabla \mathcal L=0\) → 代数移项、矩阵变形 → **直接解出最优参数固定值**    （从山顶瞬移到山脚）

- 当问题没有解析解的时候（沿着负梯度，也就是损失下降最快的方向更新参数）：

求导→ **获取下降方向**→\(\boldsymbol w_{t+1}=\boldsymbol w_t-\eta \nabla\mathcal L(\boldsymbol w_t)\)→直到梯度无限趋近于 0、损失不再下降→**无限贴近最优解，凸问题会逼近全局最优，深度网络等非凸问题通常仅收敛至局部极小值**  （从山顶顺着下坡方向一步步靠近最低点）



*梯度下降*（gradient descent）通过不断地在损失函数递减的方向上更新参数来降低误差。

（跟求后验分布求不出来的时候，用MCMC和VI推断去近似的思想一样。）



### ==没有闭式解析解时，怎样求解最优参数？==

前面已经定义了损失函数。接下来需要解决的问题是：

> 怎样找到一组参数，使损失尽可能小？

对于少数结构较简单的问题，例如普通线性回归，可以通过数学推导直接写出最优参数。这种可以用有限次代数运算直接得到的答案，称为**闭式解析解**。

- **有闭式解时：**

  对损失函数求导$\rightarrow$ 找到梯度为0 的候选点$\rightarrow$ 通过代数变形直接解出参数。

  对于线性回归的平方损失这类凸优化问题，求得的解就是全局最优解。

- **没有闭式解析解时：**

  深度神经网络的损失函数通常非常复杂，难以直接解出最优参数。因此，通常使用**梯度下降**等迭代优化方法。

  梯度表示：在当前参数位置附近，损失增加最快的方向。因此，沿着负梯度方向更新参数，通常可以使损失下降：

  $ \boldsymbol w_{t+1}=\boldsymbol w_t-\eta\nabla\mathcal L(\boldsymbol w_t).
  $

  其中，$\eta$ 是学习率，决定每次更新的步长。

  梯度下降不是一次直接算出最优参数，而是从一个初始参数出发，重复执行：
  $$
  \text{前向计算预测}
  \rightarrow
  \text{计算损失}
  \rightarrow
  \text{反向传播计算梯度}
  \rightarrow
  \text{根据梯度更新参数}.
  $$
  当训练达到预设轮数，或验证集效果不再改善时，训练停止。

需要注意：深度神经网络的损失函数通常是非凸的。梯度下降不保证一定找到全局最优解，但通常能够找到损失较低、训练稳定、泛化表现较好的参数解。

可以把两种方法简单理解为：

- **有闭式解**：通过公式直接计算最低点的位置；
- **没有闭式解**：从当前位置出发，沿着下坡方向一步步走向较低的位置。

### ==当数据量很大，梯度下降执行起来特别慢的时候怎么办？==

前面介绍的梯度下降，默认是在每次更新参数前，都使用整个训练集计算一次梯度。这种方式称为批量梯度下降（batch gradient descent）。
当数据集很大时，批量梯度下降会很慢。因为每更新一次参数，都必须先遍历全部训练样本，计算量和内存开销都较大。
因此，实际深度学习训练中通常不会每次都使用全部数据，而是每次随机选取一小批样本，利用这批样本计算梯度并更新参数。这种方法称为小批量随机梯度下降（mini-batch stochastic gradient descent）。
它的核心思想是：
不必每次都精确计算全数据集的下降方向；只要随机抽取一小批样本，就可以得到一个近似的下降方向，并据此更新参数。
每个 batch 的梯度会有一定随机波动，但计算速度更快，也更适合 GPU 并行计算。



在线性回归中的小批量随机梯度下降的步骤：

- 权重更新

\(\mathbf{w} \leftarrow \mathbf{w} - \frac{\eta}{|\mathcal{B}|}\sum_{i\in\mathcal{B}} \mathbf{x}^{(i)}\left(\mathbf{w}^\top \mathbf{x}^{(i)} + b - y^{(i)}\right)\)

- 偏置更新

\(b \leftarrow b - \frac{\eta}{|\mathcal{B}|}\sum_{i\in\mathcal{B}} \left(\mathbf{w}^\top \mathbf{x}^{(i)} + b - y^{(i)}\right)\)

不断迭代



\(\mathcal{B}\)：单次迭代随机抽取的小批量样本集合；\(|\mathcal{B}|\)：批次大小 batch size

\(\eta\)：学习率，控制每一步参数更新幅度，**超参数**，手动设定不参与训练优化

\(\partial_{(\mathbf{w},b)}l^{(i)}\)：单个样本损失对权重、偏置的偏导数（梯度）

\(\mathbf{w}^\top\mathbf{x}^{(i)}+b-y^{(i)}\)：单样本预测值与真实值的误差项



训练通常会持续多个 epoch。一个 epoch 表示模型完整看过一次训练集。

小批量训练的常见停止条件包括：

- 达到预设的训练轮数；
- 验证集损失在连续若干轮内不再改善；
- 验证集指标不再提升。

由于每个 mini-batch 的梯度存在随机波动，因此“小批量梯度接近 0”通常不是最可靠的停止标准。

### ==怎么保证小样本梯度的可靠性？==

小批量随机梯度下降每次只使用一部分样本计算梯度，因此该梯度不可能与全训练集梯度完全相同。

我们希望它至少能够大致反映全体训练数据的平均下降方向。

若每次从训练集中**均匀随机抽取**一个小批量，则该 batch 的平均梯度在期望意义下等于全训练集经验损失的梯度，即它是全数据梯度的无偏估计。

这意味着：

> 单个 batch 的梯度可能有随机波动，但经过大量随机抽样后，整体更新方向不会系统性地偏离全训练集的平均方向。



<u>如果小批量样本不具有代表性，会发生什么？</u>

若训练数据按某种规律排序，或者 batch 的构成长期存在系统偏差，则不同 batch 的梯度可能频繁偏向局部数据特征。

例如：

- 连续读取高度相似的样本时，当前 batch 的梯度可能过度反映这部分样本；
- 数据按类别排列时，某些 batch 可能几乎只包含某一个类别；
- 数据来自多个不同来源、不同时间段或不同人群时，不同 batch 可能具有明显不同的数据分布。

这可能导致：

- 梯度波动增大；
- 参数更新方向来回变化；
- 收敛变慢；
- 模型更容易优先拟合出现频率高或较早出现的数据模式。

但需要区分：

> 一个 batch 恰好只抽到某一类别，并不自动意味着梯度“有偏”。若它来自均匀随机抽样，这只是随机波动较大的一个 batch；只有当这种类别偏差长期、系统性地出现时，才会导致训练方向持续偏离整体目标。



<u>怎样减少小批量梯度的不稳定性？</u>

1. 全局打乱数据顺序

对于样本之间可以视为独立、且不需要保留原始顺序的任务，通常在每个 epoch 开始前打乱训练数据。

这样做不能让数据真正变成 IID，但可以避免“前面连续是 A 类、后面连续是 B 类”造成的 batch 偏差，使每个 batch 更接近整体训练集的组成。

2. 合理设置 batch size

batch 太小时，梯度噪声通常更大；batch 太大时，单次更新成本高、显存占用大，也可能降低更新频率。

因此，batch size 通常需要在计算效率、显存限制和梯度稳定性之间折中。

3. 对于严重类别不均衡问题，不能只依赖普通随机采样，否则模型容易被多数类主导。例如，若负样本占 99%，正样本只占 1%，模型即使几乎总是预测负类，也可能得到很高的 Accuracy，但对少数类几乎没有识别能力。

   常见处理方式包括：

   - 采样层面：类别均衡采样

     类别均衡采样的目标是改变不同类别样本进入 batch 的概率，使训练过程中模型能够更频繁地看到少数类样本，避免多数类长期主导梯度方向。

     它通常通过两类方式实现：

     - **过采样少数类**：提高少数类样本被抽中的概率，甚至重复使用少数类样本；
     - **欠采样多数类**：降低多数类样本被抽中的概率，只使用其中一部分多数类样本。

     在具体实现中，可以通过加权随机采样来完成上述目标：给少数类样本更高抽样权重，给多数类样本更低抽样权重，从而在训练 batch 中形成更均衡的类别比例。

   - 损失层面：加权损失函数

   加权损失函数不改变样本抽取方式，而是在损失函数中给少数类更高权重。这样，当模型在少数类样本上预测错误时，会受到更大的惩罚。

   - 数据层面：少数类数据增强

     对少数类样本进行合理增强，增加少数类样本的多样性，减少简单重复过采样带来的过拟合风险。

   - 评价层面：使用更合适的指标

     类别不均衡时，Accuracy 可能具有误导性，应更多关注 Precision、Recall、F1、PR-AUC 等指标。

   需要注意：类别均衡采样会改变训练 batch 中的类别分布，使其不再严格等同于原始数据分布；加权损失函数也会改变不同类别对训练目标的影响。因此，是否使用采样、加权损失，或者二者结合，需要根据任务目标和验证集表现来判断。



小批量梯度不需要每次都等于全数据梯度，但应尽量避免长期系统性偏向某类样本。

随机打乱、合理 batch size、类别均衡策略和正确的时序切分，都是提高小批量训练稳定性的常见方法。

### ==小批量梯度有噪声，深度模型损失又复杂，为什么训练仍然有效？==

在线性回归中，如果使用平方损失，损失函数是凸的，甚至可以写出闭式解。但深度神经网络不同。深度网络有很多层、很多参数和非线性激活函数，因此它的损失曲面通常是高维非凸的。

在这样的损失曲面中，训练过程可能遇到：

- 局部极小值；
- 鞍点；
- 平台区；
- 某些方向很陡、某些方向很平的区域；
- 由参数冗余造成的大量等价解。

因此，深度学习训练的目标通常不是严格找到训练损失的全局最小值，而是在有限计算资源下，找到一个训练稳定、验证集表现良好的参数区域。

也就是说，训练的关键不只是“逃离局部极小”，而是找到一个**泛化能力较好**的模型。



1.**优先选择验证集表现好的、相对稳定的解**

训练集损失低，不一定代表模型真的好。模型可能只是把训练样本中的偶然噪声也记住了。因此，实际训练中通常会额外准备验证集，用来评估模型在未参与训练的数据上的表现。如果一个模型：训练集损失很低；同时验证集损失也低；说明它不仅拟合了训练数据，也可能学到了一些更通用的规律。

如果一个模型：训练集损失持续下降；然后验证集损失开始上升；则说明模型可能开始过拟合。

所以判断模型好坏，不能只看训练损失，还要看验证集表现。

2.**随机初始化与小批量 SGD 有助于探索不同的参数区域**

深度网络训练通常从随机初始化开始。

不同的初始参数会让模型从损失曲面的不同位置开始下降，因此可能到达不同的参数区域。

小批量随机梯度下降每次只使用一个 batch 估计梯度，因此梯度中带有一定随机性。这种随机性会让参数更新路径出现轻微扰动。

该随机性会使参数在训练过程中产生一定扰动，从而有机会跨越较低的损失障碍，避免过早停留在某些狭窄或不稳定的区域。不过，这不是保证模型一定会跳出坏解。

学习率、批量大小、数据规模和优化器设置都会影响这种探索能力。多次独立训练时，应通过验证集选择表现最好的模型，而不是仅依据训练损失最低的模型。

3.**早停通过验证集选择合适的训练阶段**

训练时间越长，训练集损失通常越低，但验证集表现不一定一直变好。

早停的做法是在训练过程中定期计算验证集损失或验证集准确率。

如果验证集表现继续改善，就继续训练；

如果验证集表现连续若干轮没有改善，就停止训练；

最后保留验证集表现最好的那一轮参数。

训练集损失达到很低甚至接近零，并不必然意味着过拟合；是否过拟合应由训练集与验证集或测试集之间的性能差异判断。

4.**正则化和数据增强减少过拟合风险**

深度网络参数很多，表达能力很强，因此容易学到训练集中的偶然噪声。

为了让模型更倾向于学习稳定规律，可以使用正则化和数据增强。

- L2 权重衰减

L2 正则会在原本任务损失之外，额外惩罚过大的权重：

$\mathcal L_{\text{task}}
+
\frac{\lambda}{2}|\mathbf w|^2.
$

它的作用是限制参数过度膨胀，减少模型对少数特征的极端依赖，从而降低过拟合风险。

- Dropout

Dropout 在训练阶段会以一定概率随机将部分神经元输出置 0。这样，网络不能总是依赖固定的神经元组合完成预测，从而减少神经元之间过强的协同依赖。

现代框架中通常使用 inverted dropout：训练时对保留下来的激活值进行缩放，推理时关闭 Dropout，直接使用完整网络。

需要注意：Dropout 不是让每个神经元完全独立学习，而是减少模型对某些固定特征组合的过度依赖。

- 数据增强

数据增强通过对训练样本施加合理变换，增加样本多样性。

例如图像分类中，可以使用：

1. 平移；
2. 翻转；
3. 裁剪；
4. 亮度变化；
5. 轻微旋转。

前提是这些变换不能改变样本的真实语义。猫的图片轻微平移后仍然是猫，这种增强是合理的；如果裁剪后主体消失，就可能变成无效增强。

对于目标检测、分割和姿态估计等任务，图像增强后，边界框、mask 或关键点坐标也必须同步变换。

| 方法        | 作用层面       | 核心约束目标                         |
| ----------- | -------------- | ------------------------------------ |
| L2 权重衰减 | 参数权重数值   | 限制权重幅值，避免单一特征权重过大   |
| Dropout     | 网络神经元连接 | 破除神经元固定组合，禁止耦合记忆特例 |
| 数据增强    | 训练输入样本   | 扩充数据多样性，学习类别鲁棒本质特征 |

5.**优化器和学习率调度改善训练过程**

最简单的随机梯度下降（SGD）更新：

\(w_{t+1} = w_t - \underbrace{\eta \cdot g_t}_{\text{实际步长}}\)

- \(\eta\)：学习率（超参，人为设定）
- \(\eta \cdot g_t\)：这一步真正挪动的距离，叫**步长**

缺点：标准 SGD 只有单步梯度、无惯性、无分参数自适应步长；固定学习率导致陡坡震荡、平地停滞，梯度随机噪声让收敛路径曲折，对学习率超参极度敏感，处理高维复杂损失曲面效率低。



<u>可以怎么进行优化？</u>

Momentum、Adam 和 RMSprop 主要通过利用历史梯度、调整不同参数维度的更新尺度或改善条件数，提升训练速度和稳定性。它们可能帮助参数跨越一些较小的损失障碍，但不能简单地理解为“必然逃离局部极小”。

实践中，学习率及其调度策略通常非常重要。较大的早期学习率有助于探索参数空间，后期逐步降低学习率则有助于在较好的区域内稳定收敛。



（Momentum动量优化：借鉴物理惯性把**过去多步梯度做指数加权平均平滑**（更早梯度权重指数衰减，远期梯度影响越来越小），下坡时顺着惯性加速，震荡时抵消反向抖动。

解决：梯度方向来回抖动、下坡速度慢；

局限：**全局统一学习率**，不会给不同参数分配独立更新步长，只是平滑梯度方向，所有参数共用一套更新缩放倍率，不会为陡峭 / 平坦维度单独调节步长。它只优化下降的平稳性，无法解决 “陡坡震荡、平地停滞” 的问题。）



（RMSprop 优化：全不累积梯度方向，**单独存储每个参数梯度平方值的指数加权平均**（丢弃梯度正负号，只统计梯度波动幅度），同样是近期梯度权重更高、远期梯度逐步衰减。更新时用当前梯度除以梯度平方均值的均方根，实现逐参数自适应缩放步长（陡坡梯度大就压缩步伐，平地梯度小就放大步伐，让所有参数平稳收敛）。 

解决：陡峭维度步长爆炸震荡、平坦维度更新停滞。

局限：**只根据梯度幅值自适应调整各参数步长，不像 Momentum 那样显式累积梯度方向的惯性**；连续下坡时无法依靠惯性加速，面对鞍点、浅层局部极小很难自主滑出；同时仍需设定全局基础学习率）



（Adam 优化：同时维护过去多步梯度的一阶指数加权平均和梯度平方的二阶指数加权平均：前者相当于 Momentum，用于平滑更新方向、减少震荡；后者相当于 RMSprop，用于为不同参数自适应调整有效学习率。经过初始阶段的偏差修正后，Adam 会沿着较稳定的下降方向更新，并根据各参数近期梯度大小自动控制步长。     Adam 就是二者结合：同时保存梯度动量 + RMS 梯度平方累积。

局限：Adam 不是“必然更好”的全能优化器。它通常训练方便、收敛较快，但最终泛化性能不一定优于 SGD with Momentum；学习率、权重衰减、学习率调度和具体任务仍然会显著影响结果。）



（AdamW 优化：完整继承 Adam 的一阶、二阶指数加权平均机制，一阶梯度平滑项复用 Momentum 逻辑稳定下降方向，二阶梯度平方项复用 RMSprop 逻辑实现逐参数自适应步长，同样包含迭代初期偏差修正操作；

核心改进：是将权重衰减与梯度更新解耦：不把正则项混入梯度参与动量、自适应缩放计算，在梯度更新完成后单独对权重执行固定比例收缩，规避原版 Adam 搭配 L2 正则时求导后正则项 \(\lambda w\) 会混入梯度，跟着一阶、二阶动量、自适应分母一起缩放，权重约束力度忽大忽小，正则失效的问题。AdamW = Adam 核心自适应动量框架 + 解耦独立权重衰减。

局限：仅在开启 weight_decay 权重约束时体现独有优势，无权重衰减场景下和标准 Adam 行为完全一致；依旧依赖基础学习率、权重衰减系数、学习率调度等超参；虽然泛化表现普遍优于带 L2 正则的原版 Adam，但不代表在所有任务上一定胜过带动量 SGD，模型最终效果仍由数据集、网络结构、训练策略共同决定。）



6.**学习率调度帮助模型先探索、后收敛**

学习率决定每次参数更新走多远。

- 训练早期，较大的学习率有助于快速下降，并探索较大范围的参数空间；
- 训练后期，较小的学习率有助于在较好的区域内稳定收敛。

因此，实际训练中常常不是一直使用固定学习率，而是随着训练进行逐步降低学习率。

常见策略包括：

- 阶梯衰减：固定间隔轮次，学习率乘以衰减系数，阶段性断崖式下降（实现简单，但是学习率突变容易收敛不够平滑）；

- 指数衰减：每一轮 / 每一步学习率持续按指数倍数缩小，下降速度越来越快（全程平滑递减，但是后期学习率很小，梯度几乎不更新，提前停滞，比较少单独用）。；

- 余弦退火：不单调持续下降，模仿余弦曲线：先缓慢降低学习率，到训练末尾降到最低；一轮周期结束后可以重启学习率（下降平滑，后期精细微调效果好，收敛精度更高；但是比阶梯衰减复杂一点，需要设置周期长度）；

- warmup 后再衰减：（前几轮初始学习率设极小值，逐步慢慢提升到目标最大学习率，后面的轮数再使用阶梯 / 指数 / 余弦任意一种策略逐步降低 lr）。

- | 策略          | 变化规律             | 适用场景                             |
  | ------------- | -------------------- | ------------------------------------ |
  | 阶梯衰减      | 固定轮次断崖下跌     | 简单小模型，快速调试                 |
  | 指数衰减      | 每步持续指数下降     | 少单独使用，常做辅助                 |
  | 余弦退火      | 平滑余弦曲线下降     | 图像、深度学习主流，追求高精度收敛   |
  | Warmup + 衰减 | 先升 lr 热身，再衰减 | 大模型、深层网络、NLP / 视觉工业训练 |



7.**集成学习降低单一模型的不稳定性**

多次独立训练得到的模型可能位于不同的参数区域。将多个模型的预测进行平均或投票，常常能降低单个模型的偶然误差，提高预测稳定性和泛化能力。

但集成学习的代价是训练和推理成本更高。它不是寻找“唯一正确局部极小”的方法，而是通过组合多个模型减少单一模型决策的不确定性。

### ==除了优化参数更新，计算过程还能怎样加速？==

前面介绍的小批量随机梯度下降，主要是从**参数优化方式**上提高训练效率：每次不再使用整个训练集计算梯度，而是使用一个 mini-batch 来估计梯度并更新参数。

但这只解决了“每次用多少数据更新参数”的问题。即使已经使用 mini-batch，如果在一个 batch 内部仍然用 Python 的 `for` 循环逐个样本计算预测值，速度依然会很慢。

因此，深度学习训练还需要从**计算方式**上加速，也就是使用**批量计算**和**矢量化计算**。

1. <u>什么是批量计算？</u>

深度学习训练通常以 mini-batch 为单位进行。每次取一小批样本，完成：

```markdown
前向传播 → 计算损失 → 反向传播 → 参数更新
```

如果一个 batch 中有很多样本，不应该逐个样本单独计算，而应该把这些样本堆叠成一个带有 batch 维度的张量，然后一次性完成这一批样本的计算。

这种方式称为**批量计算**。

简单来说：

```markdown
逐样本计算：一个样本算一次，循环很多次
批量计算：把一批样本合在一起，一次性计算
```

2. <u>什么是矢量化计算？</u>

矢量化是批量计算的一种重要实现方式。

它的核心思想是：

> 不再用 `for` 循环逐个样本串行计算，而是把多个样本组织成矩阵或张量，用线性代数算子一次性完成计算。

以线性回归或全连接层为例，假设一个 batch 中有 B个样本，每个样本有 d 个特征，那么输入可以写成矩阵：

$X\in\mathbb{R}^{B\times d}.$

其中：

- B：batch size，表示这个 batch 中有多少个样本；
- d：特征维度，表示每个样本有多少个输入特征。

如果权重矩阵为：

$
W\in\mathbb{R}^{d\times h},
$

偏置为：

$
b\in\mathbb{R}^{h},
$

那么整个 batch 的输出可以一次性计算为：

$
Y=XW+b.
$

这里的 $Y\in\mathbb{R}^{B\times h}$，表示这个 batch 中所有样本的输出结果。

也就是说，原来需要对每个样本分别计算：

$x_iW+b$

现在可以通过一次矩阵运算同时算出整个 batch 的结果。

3. <u>为什么批量计算更快？</u>

如果使用 Python 的 `for` 循环逐个样本计算，会产生大量小规模运算。这样不仅 Python 循环本身有开销，也很难充分利用 CPU 或 GPU 的并行计算能力。

而批量计算会把很多小计算合并成一个较大的张量运算。这样，PyTorch、TensorFlow 等框架可以调用底层高度优化的计算库，在 CPU 或 GPU 上并行完成计算。

所以，批量计算通常比逐样本循环更快，尤其是在：

- batch size 较大；
- 模型较复杂；
- 使用 GPU 训练；

这些情况下，速度优势会更明显。

4. <u>mini-batch、批量计算和优化器的区别</u>

都是用来加速训练的，但它们解决的问题不同。

| 概念                    | 解决的问题                  | 核心作用                                         |
| ----------------------- | --------------------------- | ------------------------------------------------ |
| mini-batch SGD          | 每次用多少样本估计梯度      | 减少每次参数更新的计算量，提高优化效率           |
| 批量计算 / 矢量化计算   | 一个 batch 内部怎样算得更快 | 用矩阵或张量运算一次性计算多个样本，提高计算效率 |
| Momentum、RMSprop、Adam | 参数应该怎样更新            | 改善收敛速度和训练稳定性                         |

三者都可能缩短训练时间，但作用机制不同。

### ==为什么线性回归常用平方损失？==

前面已经介绍过，线性回归用一个线性函数预测连续数值：
$
\hat y=\boldsymbol w^\top \boldsymbol x+b.
$
在现实数据中，真实标签 $y$ 通常不会完全等于某个线性函数的输出。原因可能包括：
• 测量误差；
• 数据采集干扰；
• 未被纳入模型的重要特征；
• 真实关系本身并不是完全线性的；
• 数据中存在随机波动。
因此，线性回归常常写成下面这种数据生成形式：
$
y=\boldsymbol w^\top \boldsymbol x+b+\epsilon.
$
其中：
• $\boldsymbol w^\top \boldsymbol x+b$：模型能够解释的线性部分；
• $\epsilon$：模型没有解释掉的误差或噪声。
这里的噪声项并不是说我们真的知道每个样本的噪声是多少，而是承认：现实数据中通常存在无法被当前线性模型完全解释的部分。



1. 不引入噪声会怎样？

如果假设没有噪声，也就是：

$
y=\boldsymbol w^\top \boldsymbol x+b,
$

那么就等于认为所有样本都必须严格落在同一个线性关系上。

这在现实数据中通常很难成立。多数情况下，不存在一组 $\boldsymbol w,b$ 能让所有样本都被完全预测正确。

但这并不意味着线性回归“无解”。线性回归的做法是：

> 即使无法让每个样本的误差都等于 0，也要找到一组参数，使整体误差尽可能小。

这就是最小二乘法的思想。

2. 为什么常假设噪声服从高斯分布？

在线性回归的概率解释中，常假设噪声服从零均值高斯分布：

$
\epsilon\sim \mathcal N(0,\sigma^2).
$

这表示误差大多集中在 0 附近，较大的正误差或负误差出现概率较低。

这种假设有两个直观理由。

第一，很多现实误差可以看作许多微小因素叠加的结果。例如测量误差、环境变化、未观测变量、随机扰动等。在一定条件下，许多独立小扰动的总和往往可以近似看作正态分布。

第二，高斯噪声假设会自然推出平方损失。也就是说，如果假设：

$
y=\boldsymbol w^\top \boldsymbol x+b+\epsilon,\quad
\epsilon\sim \mathcal N(0,\sigma^2),
$

那么最大化数据出现的概率，就等价于最小化平方误差：

$
\sum_{i=1}^{n}
\left(
y^{(i)}-\hat y^{(i)}
\right)^2.
$

因此，平方损失不仅是一个直观的误差度量，也可以从高斯噪声假设下的极大似然估计推出。

3. 不同噪声假设会对应不同损失函数

平方损失不是唯一选择。不同的误差分布假设，会对应不同的优化目标。

| 噪声或输出假设 | 常见损失函数  | 特点                       |
| -------------- | ------------- | -------------------------- |
| 高斯噪声       | MSE，平方误差 | 对大误差惩罚更重，优化方便 |
| 拉普拉斯噪声   | MAE，绝对误差 | 对异常值相对更稳健         |
| 伯努利分布     | 二分类交叉熵  | 用于二分类概率建模         |
| 类别分布       | 多分类交叉熵  | 用于多分类概率建模         |

例如，如果误差中存在很多极端异常值，MSE 会因为平方项而过度放大这些异常点的影响。此时，MAE可能更合适。

MAE 对应的是：

$
\sum_{i=1}^{n}
\left|
y^{(i)}-\hat y^{(i)}
\right|.
$

它对异常值没有平方误差那么敏感，但在误差为 0 的位置不可导，优化时通常没有普通最小二乘那样简单的闭式解。

4. 为什么 MSE 在线性回归中很常用？

MSE 在线性回归中常用，主要有几个原因：

- **直观**：预测值和真实值差得越远，损失越大；
- **对大误差惩罚更重**：平方项会放大较大的预测错误；
- **数学性质好**：在线性回归中，平方损失是凸函数，便于优化；
- **求导方便**：梯度形式简单，适合梯度下降；
- **可得到闭式解**：在一定条件下，可以通过正规方程直接求解；
- **计算高效**：平方误差可以方便地写成矩阵运算，适合矢量化计算。

### ==怎么从零实现线性回归？==

任务：使用这个有限样本的数据集来恢复这个模型的参数



<u>①数据集生成：构造一个带噪声的人为数据集</u>

生成一个包含1000个样本的数据集，每个样本包含从标准正态分布中采样的2个特征。

合成数据集是一个矩阵 $$ \mathbf{X} \in \mathbb{R}^{1000 \times 2} $$ ，使用线性模型参数$\mathbf{w} = [2,\, -3.4]^\top$、$b = 4.2$ 和噪声项$\epsilon$生成数据集及其标签： $$ \mathbf{y} = \mathbf{X}\mathbf{w} + b + \epsilon. \tag{3.2.1} $$ 

代码：

```python
def synthetic_data(w, b, num_examples): 
    """生成y=Xw+b+噪声"""
    X = torch.normal(0, 1, (num_examples, len(w)))   # 从标准正态采样，生成1000*2的样本
    y = torch.matmul(X, w) + b                       # 矩阵乘法
    y += torch.normal(0, 0.01, y.shape)              # 添加均值为0，方差为0.01的高斯噪声
    return X, y.reshape((-1, 1))                     # 把y转成列向量

# 定义真实参数
true_w = torch.tensor([2, -3.4])
true_b = 4.2
# 生成数据集
features, labels = synthetic_data(true_w, true_b, 1000)
```

<u>②读取数据集：</u>

对数据集进行遍历，每次抽取一小批量样本，并使用它们来更新我们的模型。 由于这个过程是训练机器学习算法的基础，所以有必要定义一个函数， 该函数能打乱数据集中的样本并以小批量方式获取数据。

前面提到我们需要批量使用数据来更新参数，所以需要从生成的数据中分批读取：

```python
def data_iter(batch_size, features, labels):
    num_examples = len(features)
    indices = list(range(num_examples))
    random.shuffle(indices)                                 # 打乱全部样本顺序
    for i in range(0, num_examples, batch_size):            # 分批次
        batch_indices = torch.tensor(                       # 切出当前批次的下标，同时min防止最后一批不足一个 batch 时越界
            indices[i: min(i + batch_size, num_examples)])
        yield features[batch_indices], labels[batch_indices] # 返回当前批次索引对应的样本特征和标签
```

<u>③参数优化：</u>通过小批量随机梯度下降优化参数。

首先需要给参数更新的起点，即初始化。

```python
w = torch.normal(0, 0.01, size=(2,1), requires_grad=True)   # 通过从均值为0、标准差为0.01的正态分布中采样随机数来初始化权重
b = torch.zeros(1, requires_grad=True)                      # 并将偏置初始化为0
```

<u>④定义模型：用什么模型去拟合数据</u>

```python
def linreg(X, w, b):  
    """线性回归模型"""
    return torch.matmul(X, w) + b
```

<u>⑤定义损失函数：定义拟合程度的衡量标准</u>

使用均方损失

```python
def squared_loss(y_hat, y): 
    """均方损失"""
    return (y_hat - y.reshape(y_hat.shape)) ** 2 / 2
```

<u>⑥定义优化算法：使用小批量随机梯度下降</u>

```python
def sgd(params, lr, batch_size):  
    """小批量随机梯度下降"""
    with torch.no_grad():                            # 关闭自动微分计算图：参数更新属于纯数值修改，不需要记录梯度
        for param in params:
            param -= lr * param.grad / batch_size    # 使用单样本平均梯度
            param.grad.zero_()                       # 梯度清零，避免下一批次梯度累加
```

<u>⑦训练，更新参数</u>

```python
lr = 0.03                                # 学习率，控制参数每次更新的步长
num_epochs = 3                           # 完整遍历全部数据集 3 次
net = linreg
loss = squared_loss                      

for epoch in range(num_epochs):
    for X, y in data_iter(batch_size, features, labels):   # 小批量读取数据
        l = loss(net(X, w, b), y)                          # 计算当前批次损失。
        l.sum().backward()                                 # 因为l形状是(batch_size,1)，而不是一个标量。l中的所有元素被加到一起，并以此计算关于[w,b]的梯度
        sgd([w, b], lr, batch_size)                        # 调用手动实现的 SGD 优化函数，使用参数的梯度更新参数
    with torch.no_grad():                                  # 关闭梯度计算
        train_l = loss(net(features, w, b), labels)        # 查看每个epoch的loss，评估拟合效果
        print(f'epoch {epoch + 1}, loss {float(train_l.mean()):f}')
```

### ==什么是softmax回归？==

前面用线性神经网络实现了回归（预测值）的问题，同时也可以用来分类（预测类别）的问题。



生活中我们经常遇到：

- 某个电子邮件是否属于垃圾邮件文件夹？
- 某个用户可能*注册*或*不注册*订阅服务？
- 某个图像描绘的是驴、狗、猫、还是鸡？
- 某人接下来最有可能看哪部电影？



比如预测一个2X2的灰度图像属于猫、鸡、狗中的哪一个？

用一个标量表示每个像素值，每个图像有4个特征，$x_1、x_2、x_3、x_4$

对于无序类别的标注，一般采用独热编码。独热编码是一个向量，它的分量和类别一样多。

类别对应的分量设置为 1，其他所有分量设置为 0。在这个例子中，标签$y$将是一个三维向量，其中$(1,0,0)$对应于“猫”、$(0,1,0)$对应于“鸡”、$(0,0,1)$对应于“狗”： $$ y \in \{(1,0,0),(0,1,0),(0,0,1)\}. \tag{3.4.1} $$

在我们的例子中，由于我们有4个特征和3个可能的输出类别，我们将需要12个标量来表示权重（带下标的$w$），3个标量来表示偏置（带下标的$b$）。下面我们为每个输入计算三个未规范化的预测：$o_1$、$o_2$和$o_3$。 $$ \begin{aligned} o_1 &= x_1 w_{11} + x_2 w_{12} + x_3 w_{13} + x_4 w_{14} + b_1, \\ o_2 &= x_1 w_{21} + x_2 w_{22} + x_3 w_{23} + x_4 w_{24} + b_2, \\ o_3 &= x_1 w_{31} + x_2 w_{32} + x_3 w_{33} + x_4 w_{34} + b_3. \end{aligned} \tag{3.4.2} $$

![image-20260708165210472](C:\Users\s1339\AppData\Roaming\Typora\typora-user-images\image-20260708165210472.png)

要将输出视为概率，我们必须保证在任何数据上的输出都是非负的且总和为1。 

如何实现？

$softmax$函数能够将未规范化的预测变换为非负数并且总和为1，同时让模型保持可导的性质。 

为了确保最终输出的概率值总和为1，我们再让每个求幂后的结果除以它们的总和。如下式： $$ \hat{\mathbf{y}} = \text{softmax}(\mathbf{o}) \quad \text{其中} \quad \hat{y}_j = \frac{\exp(o_j)}{\sum_k \exp(o_k)} \tag{3.4.3} $$ 这里，对于所有的$j$总有$0 \leq \hat{y}_j \leq 1$。

因此，$\hat{\mathbf{y}}$可以视为一个正确的概率分布。*softmax运算不会改变未规范化的预测$\mathbf{o}$之间的大小次序，只会确定分配给每个类别的概率。*

因此，在预测过程中，我们仍然可以用下式来选择最有可能的类别。 $$ \mathop{\arg\max}_{j} \hat{y}_j = \mathop{\arg\max}_{j} o_j. \tag{3.4.4} $$

```python
def softmax(X):
    X_exp = torch.exp(X)
    partition = X_exp.sum(1, keepdim=True)
    return X_exp / partition                     # 这里应用了广播机制
```

③定义模型

```python
def net(X):
    return softmax(torch.matmul(X.reshape((-1, W.shape[0])), W) + b)
```

④定义损失函数

```python
# 模拟2个样本、3分类场景，如何一次性批量取出每个样本真实标签对应的预测概率
y = torch.tensor([0, 2])    # 前两个样本的真实类别，第一个样本真实是第一类，第二个样本真实是第三类
y_hat = torch.tensor([[0.1, 0.3, 0.6], [0.3, 0.2, 0.5]])  # 预测概率矩阵
y_hat[[0, 1], y]

cross_entropy(y_hat, y)
```

⑤定义分类精度

```python
def accuracy(y_hat, y):    
    """计算预测正确的数量"""
    if len(y_hat.shape) > 1 and y_hat.shape[1] > 1: # 判断输入y_hat是不是多分类概率矩阵
        y_hat = y_hat.argmax(axis=1) # 每行找概率最大的类别索引（预测标签）
    cmp = y_hat.type(y.dtype) == y   # 把预测值转成和y一样的数据类型，再和真实标签对比，得到布尔张量
    return float(cmp.type(y.dtype).sum())   # 布尔转数值，求和得到本批次预测正确的样本总数

accuracy(y_hat, y) / len(y)   # 分类精度即正确预测数量与总预测数量之比
```

⑥评估在数据集上的精度

```python
class Accumulator:  
    """在n个变量上累加计数"""
    def __init__(self, n):
        self.data = [0.0] * n

    def add(self, *args):
        self.data = [a + float(b) for a, b in zip(self.data, args)]

    def reset(self):
        self.data = [0.0] * len(self.data)

    def __getitem__(self, idx):
        return self.data[idx]

def evaluate_accuracy(net, data_iter):   # 计算数据集的整体精度
    """计算在指定数据集上模型的精度"""
    if isinstance(net, torch.nn.Module):
        net.eval()           # 将模型设置为评估模式
    metric = Accumulator(2)  # 得到累计的正确预测数、预测总数
    with torch.no_grad():
        for X, y in data_iter:
            metric.add(accuracy(net(X), y), y.numel())  #  累加：本批次正确数，本批次总样本数   (y.numel()返回y元素总个数=batch_size)
    return metric[0] / metric[1]

evaluate_accuracy(net, test_iter)
```

⑦训练模型

```python
def train_epoch_ch3(net, train_iter, loss, updater):  
    """训练模型一个迭代周期（定义见第3章）"""
    # 将模型设置为训练模式
    if isinstance(net, torch.nn.Module):
        net.train()
    # 训练损失总和、训练准确总和、总样本数
    metric = Accumulator(3)
    for X, y in train_iter:
        # 计算梯度并更新参数
        y_hat = net(X)
        l = loss(y_hat, y)
        if isinstance(updater, torch.optim.Optimizer):
            # 使用PyTorch内置的优化器和损失函数
            updater.zero_grad()
            l.mean().backward()
            updater.step()
        else:
            # 使用定制的优化器和损失函数
            l.sum().backward()
            updater(X.shape[0])
        metric.add(float(l.sum()), accuracy(y_hat, y), y.numel())
    return metric[0] / metric[2], metric[1] / metric[2]  # 得到本轮平均训练损失和训练集准确率

def train_ch3(net, train_iter, test_iter, loss, num_epochs, updater):  
    """训练模型"""
    animator = Animator(xlabel='epoch', xlim=[1, num_epochs], ylim=[0.3, 0.9],
                        legend=['train loss', 'train acc', 'test acc'])
    # 循环遍历每一轮训练
    for epoch in range(num_epochs):
        # 执行1轮完整训练，返回(本轮平均损失, 本轮训练集平均精度)
        train_metrics = train_epoch_ch3(net, train_iter, loss, updater)
        # 在测试集上计算模型精度（不更新参数，仅评估）
        test_acc = evaluate_accuracy(net, test_iter)
        # 向动态图添加当前轮数据：x=epoch+1，y=(训练损失,训练精度,测试精度)
        animator.add(epoch + 1, train_metrics + (test_acc,))
    train_loss, train_acc = train_metrics
    # 模型必须达到基础合格标准，否则抛出异常提示
    assert train_loss < 0.5, train_loss   # 训练损失必须小于0.5
    assert train_acc <= 1 and train_acc > 0.7, train_acc # 训练精度在0.7~1之间
    assert test_acc <= 1 and test_acc > 0.7, test_acc    # 测试精度在0.7~1之间
    
lr = 0.1
```
